◆ One-time setup

git --version
check Git is installed
git config --global user.name "Name"
git config --global user.email "you@…"
stamped on every commit
git config --global core.autocrlf true
line endings · Mac/Linux use input
git config --global init.defaultBranch main
ssh -T git@git.selangor.gov.my
test GitLab access

◆ Start a repository

git init
turn the current folder into a repo
git clone <url>
copy an existing repo down, history and all

◆ The daily loop

Working—add→ Staging—commit→ Local—push→ GitLab
git pull
get the latest before you start
git status
what changed & where it sits
git add <file> · git add .
stage one file · or everything
git commit -m "message"
save a snapshot
git push
share it to GitLab

◆ Inspect & compare

git log --oneline
history, compact
git status -s
short status
git diff
unstaged changes
git diff --staged
what's staged for commit
git show <hash>
one commit in full

◆ Branching & merging

git switch -c <name>
create a branch & jump on
git switch main
move between branches
git branch · -d <name>
list · delete a merged branch
git merge <name>
fold a branch into current
git merge --abort
back out of a messy merge
Name for the job: feature/… bugfix/…

◆ Resolve a conflict

Pull & see the flag: git pull
Open the file, find <<< === >>>
Edit to the final text you want
Delete all three marker lines
git add <file> then git commit
Stuck? git merge --abort rewinds. Nothing lost.

◆ Undo & recover

git restore <file>
discard edits, back to last commit
git restore -s HEAD~1 <file>
bring back an earlier version
git restore --staged <file>
unstage, keep your edits
git commit --amend
fix last message (before push)
git reset --soft HEAD~1
undo last commit, keep work
git revert <hash>
reverse a pushed commit · team-safe
git reflog
find a "lost" commit's hash
git clean -fd
delete untracked files · no undo

◆ GitLab collaboration

git remote -v
see your remotes (origin = default)
git push -u origin <branch>
push a new branch, print an MR link
Branch→Push→ Open MR→Review→Merge
Small MRs · real description · link the issue · review kindly & fast · delete merged branches.

◆ .gitignore

node_modules/ vendor/ dist/
deps & build output
.env *.key .DS_Store
secrets & OS clutter — never commit
git rm -r --cached <path>
stop tracking something already committed
Leaked a secret? Rotate it — .gitignore won't erase past commits.
Pull before you start, push when you pause. Branch for every change. Unsure? Run git status. Messy? STOP & ask before piling on.
